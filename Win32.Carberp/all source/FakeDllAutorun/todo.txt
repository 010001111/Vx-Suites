1) установка
  по команде ДЛЛка грузится и запускается ф-ция установки
    
  1) для FF, Opera и IE находятся dll-ки, которые следует подменить
  2) В тело нашей длл вшивается имя новой длл, которую нужно подгружать,
     прибиваются все процессы, длл переименовывается, вместо оригинальной вставляется наша
  
2) процесс подгрузки
   Идея- при загрузке заменить хендл нашей длл посредством 
   подмены адреса возврата ф-ции LoadLibraryExW своим загрузчиком, который прогрузит в свою очередь 
   нашу дллку 
  
Если адрес подгружаемой длл не забит - то не подгружаем
  
Для корректной работы с ф-циями модулей (LoadLibrary, GetModuleHandle ) обмениваются имена загруженных Dll 
в PEB

1) Инсталятор должен
   - загружать bot.plug со встроенными данными для соединения
   - сохранять зашифрованным в случайном месте
   - экспортировать ф-цию для установки в разные места (принимает строку со списком куда)
     - подготовить дллку для сохранения (встроить новое имя длл, адрес)
     - поменять имя дллки для приложения
     - сохранить авторановскую длл
   
     
2) Авторан должен
   - загружать с подменой себя на оригинальную
   - получать с диска сохранённый инсталером bot.plug
   - дешифровать, загружать его в памяти и вызывать для него ф-ция Start
